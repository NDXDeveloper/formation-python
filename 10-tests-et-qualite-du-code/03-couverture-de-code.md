# 10.3 : Couverture de code

## Introduction

Imaginez que vous √™tes architecte et que vous venez de terminer les plans d'une maison. Avant de commencer la construction, vous voulez v√©rifier que tous les aspects ont √©t√© pens√©s : la plomberie, l'√©lectricit√©, la ventilation, l'isolation... La **couverture de code** fonctionne de la m√™me mani√®re : elle vous indique quelles parties de votre code ont √©t√© "visit√©es" par vos tests et lesquelles ont √©t√© oubli√©es.

C'est un outil pr√©cieux pour s'assurer que vos tests examinent bien tout votre code, mais attention : 100% de couverture ne garantit pas 100% de qualit√© !

## Qu'est-ce que la couverture de code ?

### D√©finition simple
La couverture de code mesure le pourcentage de votre code qui est ex√©cut√© pendant les tests. Si votre programme fait 100 lignes et que vos tests en ex√©cutent 80, vous avez 80% de couverture.

### Exemple visuel
```python
def calculer_remise(prix, age_client):
    """Calcule la remise selon l'√¢ge du client"""
    if prix < 0:                    # Ligne 1 ‚úÖ Test√©e
        raise ValueError("Prix invalide")  # Ligne 2 ‚úÖ Test√©e

    if age_client < 18:             # Ligne 3 ‚úÖ Test√©e
        return prix * 0.9           # Ligne 4 ‚ùå PAS test√©e
    elif age_client >= 65:          # Ligne 5 ‚úÖ Test√©e
        return prix * 0.8           # Ligne 6 ‚ùå PAS test√©e
    else:
        return prix                 # Ligne 7 ‚úÖ Test√©e

# Test existant (incomplet)
def test_calculer_remise():
    # Cas normal
    assert calculer_remise(100, 30) == 100

    # Cas d'erreur
    try:
        calculer_remise(-10, 30)
    except ValueError:
        pass

# Couverture : 4 lignes test√©es sur 6 = 67%
```

### Types de couverture

#### 1. Couverture de lignes (Line Coverage)
Mesure quelles lignes de code ont √©t√© ex√©cut√©es.

#### 2. Couverture de branches (Branch Coverage)
Mesure quelles branches des conditions ont √©t√© prises.

```python
def evaluer_note(note):
    if note >= 10:        # Branche: True ET False test√©es ?
        return "R√©ussi"   # Ex√©cut√© si note >= 10
    else:
        return "√âchou√©"   # Ex√©cut√© si note < 10

# Pour 100% de couverture de branches, il faut tester :
# - Un cas o√π note >= 10 (branche True)
# - Un cas o√π note < 10 (branche False)
```

#### 3. Couverture de fonctions (Function Coverage)
Mesure quelles fonctions ont √©t√© appel√©es au moins une fois.

```python
# user_manager.py
class User:
    def __init__(self, nom, email, age):
        self.nom = nom
        self.email = email
        self.age = age
        self.actif = True

    def est_adulte(self):
        return self.age >= 18

    def est_senior(self):
        return self.age >= 65

class UserManager:
    def __init__(self):
        self.users = {}
        self.next_id = 1

    def ajouter_user(self, nom, email, age):
        """Ajoute un utilisateur"""
        if not nom or not email:
            raise ValueError("Nom et email requis")

        if age < 0:
            raise ValueError("L'√¢ge ne peut pas √™tre n√©gatif")

        if age > 150:
            raise ValueError("L'√¢ge ne peut pas d√©passer 150 ans")

        # V√©rifier si l'email existe d√©j√†
        for user in self.users.values():
            if user.email == email:
                raise ValueError("Email d√©j√† utilis√©")

        user = User(nom, email, age)
        user_id = self.next_id
        self.users[user_id] = user
        self.next_id += 1
        return user_id

    def obtenir_user(self, user_id):
        """R√©cup√®re un utilisateur"""
        return self.users.get(user_id)

    def supprimer_user(self, user_id):
        """Supprime un utilisateur"""
        if user_id in self.users:
            del self.users[user_id]
            return True
        return False

    def desactiver_user(self, user_id):
        """D√©sactive un utilisateur"""
        user = self.users.get(user_id)
        if user:
            user.actif = False
            return True
        return False

    def obtenir_users_actifs(self):
        """Retourne les utilisateurs actifs"""
        return [user for user in self.users.values() if user.actif]

    def obtenir_users_adultes(self):
        """Retourne les utilisateurs adultes"""
        return [user for user in self.users.values() if user.est_adulte()]

    def obtenir_statistiques(self):
        """Retourne des statistiques"""
        total = len(self.users)
        if total == 0:
            return {"total": 0, "adultes": 0, "seniors": 0, "actifs": 0}

        adultes = len([u for u in self.users.values() if u.est_adulte()])
        seniors = len([u for u in self.users.values() if u.est_senior()])
        actifs = len([u for u in self.users.values() if u.actif])

        return {
            "total": total,
            "adultes": adultes,
            "seniors": seniors,
            "actifs": actifs
        }
```

## R√©sum√©

La couverture de code est un outil pr√©cieux mais qui doit √™tre utilis√© intelligemment :

### **Points cl√©s :**
- **Mesure** : Pourcentage de code ex√©cut√© par les tests
- **Types** : Lignes, branches, fonctions
- **Outil** : coverage.py avec pytest-cov
- **Objectif** : 80-90% de couverture pour la plupart des projets

### **Bonnes pratiques :**
- **Qualit√© avant quantit√©** : 80% de tests excellents valent mieux que 100% de tests m√©diocres
- **Focus sur le critique** : Priorisez les parties importantes de votre code
- **Exclusions raisonnables** : Utilisez `pragma: no cover` pour le code de debug/logging
- **Couverture de branches** : Plus importante que la simple couverture de lignes
- **Int√©gration CI/CD** : Automatisez la v√©rification de couverture

### **M√©triques utiles :**
- **80-90%** : Couverture cible pour la plupart des projets
- **95%+** : Pour le code critique (paiements, s√©curit√©)
- **Moins important** : Code de logging, debug, scripts one-shot

### **Workflow recommand√© :**
1. **√âcrire** les tests pour les cas principaux
2. **Mesurer** la couverture initiale
3. **Identifier** les parties manquantes importantes
4. **Compl√©ter** les tests pour ces parties
5. **Exclure** le code non critique si n√©cessaire
6. **Automatiser** la v√©rification en CI/CD

### **Outils essentiels :**
- `coverage run` : Ex√©cuter avec mesure
- `coverage report` : Rapport textuel
- `coverage html` : Rapport visuel
- `pytest --cov` : Int√©gration avec pytest
- Configuration `.coveragerc` : Personnalisation

### **Signaux d'alarme :**
- Couverture qui diminue soudainement
- Nouveau code sans tests
- Branches importantes non test√©es
- Tests qui passent mais ne testent rien de significatif
