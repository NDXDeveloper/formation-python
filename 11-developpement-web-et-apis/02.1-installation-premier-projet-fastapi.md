# 11.2.1 Installation et premier projet FastAPI

## Introduction

FastAPI est un framework web moderne et performant pour créer des APIs avec Python. Dans cette section, vous allez apprendre à installer FastAPI et à créer votre tout premier projet. Nous allons procéder étape par étape pour que tout soit clair, même si vous débutez.

## Pourquoi FastAPI ?

Avant de commencer, rappelons rapidement pourquoi FastAPI est un excellent choix :

- ⚡ **Rapide** : Performances comparables à Node.js et Go
- 🎯 **Moderne** : Utilise les fonctionnalités récentes de Python (type hints, async/await)
- 📝 **Documentation automatique** : Interface Swagger générée automatiquement
- ✅ **Validation** : Vérification automatique des données avec Pydantic
- 🔰 **Facile à apprendre** : Syntaxe intuitive et documentation excellente

## Prérequis

Avant de commencer, assurez-vous d'avoir :

- **Python 3.10 ou supérieur** installé sur votre ordinateur
- Des connaissances de base en Python (variables, fonctions, décorateurs)
- Un éditeur de code (VS Code, PyCharm, Sublime Text, etc.)
- Un terminal ou invite de commande

Pour vérifier votre version de Python, ouvrez un terminal et tapez :

```bash
python --version
# ou
python3 --version
```

Si la version affichée est 3.10 ou supérieure, vous êtes prêt !

## Étape 1 : Créer un environnement virtuel

Il est fortement recommandé de créer un **environnement virtuel** pour chaque projet Python. Cela permet d'isoler les dépendances de votre projet et d'éviter les conflits entre différentes versions de bibliothèques.

### Qu'est-ce qu'un environnement virtuel ?

Un environnement virtuel est comme une "bulle" isolée où vous pouvez installer des packages Python sans affecter votre installation Python système. Chaque projet peut avoir son propre environnement avec ses propres versions de bibliothèques.

### Créer l'environnement virtuel

Ouvrez votre terminal et naviguez vers le dossier où vous souhaitez créer votre projet :

```bash
# Créer un dossier pour votre projet
mkdir mon_premier_fastapi  
cd mon_premier_fastapi  

# Créer l'environnement virtuel
python -m venv venv
```

Cette commande crée un dossier `venv` contenant votre environnement virtuel.

### Activer l'environnement virtuel

Maintenant, activez l'environnement virtuel :

**Sur Windows :**
```bash
venv\Scripts\activate
```

**Sur macOS/Linux :**
```bash
source venv/bin/activate
```

Vous devriez voir le nom de votre environnement virtuel `(venv)` apparaître au début de votre ligne de commande. Cela signifie que l'environnement est activé !

```bash
(venv) C:\Users\VotreNom\mon_premier_fastapi>
```

## Étape 2 : Installer FastAPI et Uvicorn

Maintenant que votre environnement virtuel est activé, vous pouvez installer FastAPI et ses dépendances.

### Qu'est-ce qu'Uvicorn ?

**Uvicorn** est un serveur ASGI (Asynchronous Server Gateway Interface). En termes simples, c'est le programme qui va faire tourner votre application FastAPI et gérer les requêtes HTTP.

Pensez à Uvicorn comme le moteur qui fait fonctionner votre voiture (FastAPI).

### Installation

Installez FastAPI et Uvicorn avec pip :

```bash
pip install fastapi uvicorn[standard]
```

Cette commande installe :
- `fastapi` : Le framework FastAPI lui-même
- `uvicorn[standard]` : Le serveur ASGI avec des fonctionnalités supplémentaires optimales

L'installation peut prendre quelques instants. Une fois terminée, vous êtes prêt à créer votre première application !

### Vérifier l'installation

Pour vérifier que l'installation s'est bien déroulée :

```bash
pip list
```

Vous devriez voir `fastapi` et `uvicorn` dans la liste des packages installés.

## Étape 3 : Créer votre première application FastAPI

Créez un nouveau fichier Python nommé `main.py` dans votre dossier de projet. C'est dans ce fichier que vous allez écrire le code de votre API.

### Le code minimal

Ouvrez `main.py` et écrivez ce code :

```python
from fastapi import FastAPI

# Créer une instance de l'application FastAPI
app = FastAPI()

# Définir une route simple
@app.get("/")
def lire_racine():
    return {"message": "Bienvenue sur mon API FastAPI !"}
```

C'est tout ! Vous avez créé votre première API. Impressionnant, non ? Mais attendez, voyons ce que fait exactement ce code.

### Explication ligne par ligne

```python
from fastapi import FastAPI
```
Cette ligne importe la classe `FastAPI` depuis le module `fastapi`. C'est la classe principale que nous allons utiliser pour créer notre application.

```python
app = FastAPI()
```
Ici, nous créons une **instance** de FastAPI. Cette instance `app` représente votre application web. Toutes les routes et configurations seront attachées à cet objet.

```python
@app.get("/")
```
C'est un **décorateur** qui définit une route. Décomposons-le :
- `@app.get` : Indique que cette fonction répondra aux requêtes HTTP GET
- `"/"` : C'est le chemin de la route (l'URL). `/` signifie la racine de votre site (exemple : `http://localhost:8000/`)

```python
def lire_racine():
    return {"message": "Bienvenue sur mon API FastAPI !"}
```
C'est la fonction qui sera exécutée quand quelqu'un accède à cette route. Elle retourne un dictionnaire Python qui sera automatiquement converti en JSON par FastAPI.

Le résultat sera :
```json
{
    "message": "Bienvenue sur mon API FastAPI !"
}
```

### Structure du projet

À ce stade, votre projet devrait avoir cette structure :

```
mon_premier_fastapi/
│
├── venv/              # Environnement virtuel (ne pas modifier)
│
└── main.py            # Votre application FastAPI
```

## Étape 4 : Lancer votre application

Maintenant, lançons votre application pour voir si tout fonctionne !

Dans votre terminal (avec l'environnement virtuel activé), exécutez :

```bash
uvicorn main:app --reload
```

### Décortiquons cette commande

- `uvicorn` : Le serveur ASGI qui va faire tourner votre application
- `main` : Le nom de votre fichier Python (sans le `.py`)
- `app` : Le nom de la variable FastAPI dans votre fichier
- `--reload` : Active le rechargement automatique. Quand vous modifiez votre code, le serveur redémarre automatiquement. **Très pratique en développement !**

### Que se passe-t-il ?

Vous devriez voir quelque chose comme :

```
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)  
INFO:     Started reloader process [12345] using StatReload  
INFO:     Started server process [67890]  
INFO:     Waiting for application startup.  
INFO:     Application startup complete.  
```

Félicitations ! 🎉 Votre serveur est lancé et écoute sur `http://127.0.0.1:8000`.

## Étape 5 : Tester votre API

Il y a plusieurs façons de tester votre API. Voyons les principales.

### Méthode 1 : Le navigateur web

Ouvrez votre navigateur et allez à l'adresse :

```
http://127.0.0.1:8000/
```

ou

```
http://localhost:8000/
```

Vous devriez voir s'afficher :

```json
{"message": "Bienvenue sur mon API FastAPI !"}
```

C'est la réponse JSON retournée par votre API !

### Méthode 2 : La documentation interactive (Swagger)

FastAPI génère **automatiquement** une documentation interactive pour votre API. C'est l'une de ses fonctionnalités les plus appréciées !

Allez à l'adresse :

```
http://127.0.0.1:8000/docs
```

Vous verrez une interface Swagger élégante qui liste toutes vos routes et vous permet de les tester directement dans le navigateur.

**Pour tester votre route :**
1. Cliquez sur la route `GET /`
2. Cliquez sur "Try it out"
3. Cliquez sur "Execute"
4. Vous verrez la réponse de votre API

### Méthode 3 : Documentation alternative (ReDoc)

FastAPI fournit également une autre interface de documentation. Allez à :

```
http://127.0.0.1:8000/redoc
```

C'est une alternative à Swagger, au design différent mais tout aussi fonctionnelle.

### Méthode 4 : Avec curl (ligne de commande)

Si vous êtes à l'aise avec la ligne de commande, vous pouvez utiliser `curl` :

```bash
curl http://127.0.0.1:8000/
```

Ou avec Python et la bibliothèque `requests` (dans un autre terminal) :

```python
import requests

response = requests.get("http://127.0.0.1:8000/")  
print(response.json())  
```

## Ajouter plus de routes

Maintenant que votre première route fonctionne, ajoutons-en d'autres pour comprendre les concepts de base.

### Route avec un message personnalisé

Modifiez votre fichier `main.py` :

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def lire_racine():
    return {"message": "Bienvenue sur mon API FastAPI !"}

@app.get("/bonjour")
def dire_bonjour():
    return {"message": "Bonjour ! Comment allez-vous ?"}

@app.get("/info")
def obtenir_info():
    return {
        "nom_application": "Mon API",
        "version": "1.0.0",
        "framework": "FastAPI"
    }
```

Sauvegardez le fichier. Grâce à l'option `--reload`, le serveur redémarre automatiquement !

Testez vos nouvelles routes :
- `http://127.0.0.1:8000/bonjour`
- `http://127.0.0.1:8000/info`

### Route avec paramètre dans l'URL

Vous pouvez également créer des routes avec des paramètres dynamiques :

```python
@app.get("/utilisateur/{nom}")
def lire_utilisateur(nom: str):
    return {"message": f"Bonjour {nom} !"}
```

Avec cette route, vous pouvez accéder à :
- `http://127.0.0.1:8000/utilisateur/Alice` → `{"message": "Bonjour Alice !"}`
- `http://127.0.0.1:8000/utilisateur/Bob` → `{"message": "Bonjour Bob !"}`

Le `{nom}` dans l'URL est un **paramètre de chemin** qui est automatiquement passé à votre fonction.

### Route avec paramètre d'âge et validation

```python
@app.get("/age/{age}")
def verifier_age(age: int):
    if age >= 18:
        return {"message": f"Vous avez {age} ans, vous êtes majeur."}
    else:
        return {"message": f"Vous avez {age} ans, vous êtes mineur."}
```

Remarquez le type `int` pour le paramètre `age`. FastAPI valide automatiquement que la valeur est bien un nombre entier. Si vous essayez d'accéder à `/age/abc`, vous recevrez une erreur de validation automatique !

## Comprendre les types de requêtes HTTP

FastAPI supporte tous les types de requêtes HTTP standards. Voici les plus courants :

### GET - Récupérer des données

```python
@app.get("/articles")
def lire_articles():
    return {"articles": ["Article 1", "Article 2"]}
```

Utilisé pour **lire** des données sans les modifier.

### POST - Créer des données

```python
from fastapi import FastAPI  
from pydantic import BaseModel  

app = FastAPI()

class Article(BaseModel):
    titre: str
    contenu: str

@app.post("/articles")
def creer_article(article: Article):
    return {"message": "Article créé", "article": article}
```

Utilisé pour **créer** de nouvelles ressources. Ne vous inquiétez pas pour `BaseModel`, nous l'expliquerons dans la prochaine section !

### PUT - Mettre à jour des données

```python
@app.put("/articles/{id}")
def modifier_article(id: int, article: Article):
    return {"message": f"Article {id} modifié"}
```

Utilisé pour **mettre à jour** complètement une ressource existante.

### DELETE - Supprimer des données

```python
@app.delete("/articles/{id}")
def supprimer_article(id: int):
    return {"message": f"Article {id} supprimé"}
```

Utilisé pour **supprimer** une ressource.

## Bonnes pratiques pour débuter

Voici quelques conseils pour bien démarrer avec FastAPI :

### 1. Utilisez toujours un environnement virtuel

Cela évite les conflits de dépendances et garde votre système Python propre.

### 2. Utilisez --reload en développement

Le flag `--reload` permet au serveur de redémarrer automatiquement quand vous modifiez votre code.

### 3. Consultez la documentation interactive

L'interface Swagger (`/docs`) est votre meilleure amie pour tester vos routes rapidement.

### 4. Utilisez les type hints

FastAPI utilise les annotations de type Python pour la validation. Habituez-vous à les utiliser :

```python
def ma_fonction(nom: str, age: int) -> dict:
    return {"nom": nom, "age": age}
```

### 5. Organisez votre code

Pour de petits projets, un seul fichier `main.py` suffit. Mais pour des projets plus importants, pensez à organiser votre code en modules.

### 6. Gardez le serveur en arrière-plan

Vous pouvez ouvrir plusieurs terminaux : un pour le serveur, un autre pour tester vos requêtes.

## Résolution de problèmes courants

### Erreur : "Port already in use"

Le port 8000 est déjà utilisé. Utilisez un autre port :

```bash
uvicorn main:app --reload --port 8001
```

### Erreur : "Module 'fastapi' has no attribute..."

Vérifiez que vous avez bien activé votre environnement virtuel et que FastAPI est installé :

```bash
pip list | grep fastapi
```

### Le serveur ne redémarre pas automatiquement

Vérifiez que vous utilisez bien le flag `--reload` :

```bash
uvicorn main:app --reload
```

### Erreur 404 sur une route

Vérifiez l'orthographe de votre URL et assurez-vous que la route est bien définie dans votre code.
